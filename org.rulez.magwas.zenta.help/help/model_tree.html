The Model Tree

A model consists of a number of elements.

Each element in the model can connect to one or more other elements via one or more relationships (connections).

Until you create a template view (or load a model with it), you will have only one type of element and relationship. You can define other types with creating metamodels. A metamodel is just a view with a property called "template"

A model consists of configurations of these elements connected to each other via the various relationships. The normative model is represented in Zenta in the "Models" window as a tree structure organised into folders:


The Model Tree window showing an example model

Each element and relationship is placed in a hierarchy of folders in the Model Tree.
